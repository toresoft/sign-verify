# Signature verification (docs/en/05)
> _Original: `docs/en/05-signature-verification.md` (~16 KB)_

## Summary

The user-facing guide to **how clients verify signatures** against sign-verify-2. Covers the validation pipeline and report types (§4.1), validation profiles with their DSS policy XML format and the central `Level` attribute concept (§4.2), the **synchronous** verification API (§4.3), and the **asynchronous** verification API with webhooks (§4.4). This is the operational counterpart to the design doc's engine section ([[sources/SRC-2026-06-27-002]] §6).

The synchronous endpoint `POST /api/v1/verifications` (multipart: `file` + optional `metadata` JSON) acquires a semaphore and delegates to the DSS adapter, returning the requested report types. The async flow (`POST …/jobs` → poll `GET …/jobs/{id}` or receive an HMAC-signed webhook) is backed by the persisted `validation_job` state machine.

## Key Takeaways

- **Reports available:** `simple`, `detailed`, `diagnostic`, `etsi`. Unknown report value → `400 unknown report type`; malformed metadata JSON → `400 invalid metadata json`.
- **Profiles (§4.2):** built-in presets + custom profiles + per-request `profileOverrides`. The DSS **policy XML** is governed by a `Level` attribute (`model`, constraints, basic/long-term archival levels) — see the "Level attribute" key concept.
- **Sync API (§4.3):** `POST /api/v1/verifications`; response carries `indication` (e.g. `TOTAL_PASSED`), `subIndication`, `signatureFormat` (e.g. `PAdES-BASELINE-B`), `signatureCount`, plus the requested report bodies. Concurrency saturation → `429 excessive-load.concurrency`.
- **Async lifecycle (§4.4):** states `PENDING`/`RUNNING`/`COMPLETED`/`FAILED`/`DELIVERED`/`DELIVERY_FAILED`/`DELETED`. Worker polls (default 5s), skips when the DSS circuit breaker is OPEN. Result retrieval `GET /api/v1/verifications/jobs/{jobId}` → owner/PRIVILEGED only (else `404`); `DELETED` → `410 Gone`.
- **Callback delivery:** webhook POST, HMAC-signed, with success/non-retryable/retryable status handling and exponential backoff.
