# sign-verify-2 — Design Document (architettura)
> _Original: `docs/superpowers/specs/2026-06-07-sign-verify-design.md` (Italian, ~48 KB)_

## Summary

The authoritative **architecture & design** specification for sign-verify-2, the Spring Boot service that verifies digital signatures under eIDAS via the [[entities/dss]] library. Defines the hexagonal-lite layered structure (§3), the data model & invariants (§4), the REST API contract (§5), the signature verification engine with profiles (§6), [[concepts/trusted-lists]] management with hot-swap refresh (§7), async jobs + HMAC callbacks (§8), load protection & multi-instance coordination (§9), and cross-cutting concerns (§10–14). This is the single source of truth for **how the system is meant to work**.

The architectural style is **Maven single-module + layered + hexagonal-lite for external integrations**: high-risk/complex integrations (DSS, OAuth IdP, callback HTTP, filesystem, password hashing) sit behind **domain ports** with **adapters** as implementations, enforced by ArchUnit. The data model is owned by Flyway; tables include `api_key`, `verification_profile`, `validation_job`, `trusted_certificate` (queryable mirror post-TSL-refresh), `tsl_refresh` (history), `audit_log` (append-only), and `shedlock`.

## Key Takeaways

- **Architecture:** `api → application → domain(model/port/exception) → adapter(dss/storage/callback/crypto) → persistence → security → config`. Ports: `SignatureValidatorPort`, `ExtractionPort`, `TslRefresherPort`, `CallbackDispatcherPort`, `DocumentStoragePort`, `PasswordHasherPort`, `SecretCipherPort`.
- **API:** base path `/api/v1/*`, JSON bodies (multipart for uploads), errors as `application/problem+json` (RFC 9457), design-first OpenAPI at `src/main/resources/openapi/openapi.yaml`, stable error codes via `urn:signverify:error:<code>`.
- **Verification engine (§6):** DSS-configured; profiles = **preset + custom + per-request override**; sync endpoint backs off with a semaphore (`429 excessive-load.concurrency`); reports: `simple`, `detailed`, `diagnostic`, `etsi`.
- **TSL management (§7):** `app.tsl.sources` list (type `LOTL`/`TL`), EU LOTL default `https://ec.europa.eu/tools/lotl/eu-lotl.xml` with pivot support, OJ keystore (`oj-keystore.p12`) as LOTL trust anchor, atomic hot-swap refresh via `TrustedListsCertificateSource`, scheduler + ShedLock (`startup-mode: BACKGROUND|BLOCKING|SKIP`).
- **Async jobs (§8):** `validation_job` state machine `PENDING→RUNNING→COMPLETED→DELIVERED` (or `FAILED`/`DELIVERY_FAILED`/`DELETED`); callback dispatch polls `FOR UPDATE SKIP LOCKED`, HMAC-signed (default `HmacSHA256`), callback secret encrypted at rest with AES-256-GCM master key.
- **Load protection (§9):** sync concurrency limit, async back-pressure, **DSS circuit breaker** (OPEN skips validation cycles), ShedLock for multi-instance scheduler coordination.
