# OpenAPI 3 contract (openapi.yaml)
> _Original: `src/main/resources/openapi/openapi.yaml` (~24 KB, design-first)_

## Summary

The **authoritative API contract** for sign-verify-2 (OpenAPI 3.0.3, `sign-verify-2 API` v1.0.0). Design-first: the OpenAPI Generator produces controller interfaces + DTOs under `api.spi`/`api.dto`, and `OpenApiContractIT` guards the contract. Base path `/api/v1/*`; security via `ApiKeyAuth` (`X-API-Key`) and optional `OAuth2Bearer`. Errors are `application/problem+json` (RFC 9457) with stable URN codes.

Resource tags: **ApiKeys**, **Profiles**, **Verifications**, **Extractions**, **Tsl**, **Audit**, **Health**.

## Key Takeaways

- **Design-first:** edit `openapi.yaml`; generator emits interfaces/DTOs; `OpenApiContractIT` enforces conformance (see design doc §5.1, [[sources/SRC-2026-06-27-002]]).
- **Tags / domains:** ApiKeys (create/list/enable-disable/delete), Profiles (validation profile CRUD), Verifications (sync + async jobs), Extractions (original file extraction), Tsl (status/refresh/cert lookup), Audit (log query), Health (`/actuator/health`, public).
- **Security schemes:** `ApiKeyAuth` (header `X-API-Key`) and `OAuth2Bearer` (Bearer JWT); endpoints declare their requirements; `/actuator/health` is public (`security: []`).
- **Invariants surfaced by the contract:** API keys returned in plaintext only once at creation; the last enabled PRIVILEGED key cannot be deleted (`409`); async job result retrieval returns `404` for non-owners and `410 Gone` for deleted jobs.
