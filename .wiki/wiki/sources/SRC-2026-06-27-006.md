# Original file extraction (docs/en/06)
> _Original: `docs/en/06-file-extraction.md` (~4 KB)_

## Summary

How to retrieve the **original unsigned document(s)** embedded inside a signed file. The endpoint `POST /api/v1/extractions` (multipart, `file` part required) delegates to the `ExtractionPort` (DSS adapter), which locates the originals and infers the signature format. A **single** original is returned directly as binary with its MIME type; **multiple** originals (typical of ASiC-E) are packed into a ZIP (`application/zip`, named `originals.zip`). Informational headers `X-Signature-Format` and `X-Document-Count` are always present.

## Key Takeaways

- **Endpoint:** `POST /api/v1/extractions` — multipart/form-data, `file` required.
- **Response shape:** single original → binary with its `Content-Type` + `Content-Disposition: attachment`; multiple → ZIP.
- **Headers:** `X-Signature-Format` (e.g. `CAdES`, `ASiC-E`), `X-Document-Count`.
- **Underlying port:** `ExtractionPort` implemented by the DSS extraction adapter ([[entities/dssextractionadapter]]), mirroring the design doc ([[sources/SRC-2026-06-27-002]] §6.6).
